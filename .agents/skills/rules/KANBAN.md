# Kanban

## Principios

- Visualizar el trabajo
- Limitar el trabajo en proceso (WIP)
- Maximizar el flujo
- Mejora continua

## Columnas

```
To Do → In Progress → Review → Done
```

Opcional:
```
Backlog → To Do → In Progress → Review → Done → Deploy
```

## WIP Limits

Limitar tareas por columna:

| Columna | WIP Limit |
|---------|-----------|
| To Do | Sin limite |
| In Progress | 3 |
| Review | 2 |
| Done | Sin limite |

## Reglas

1. **No mover tarea si no hay espacio** - Si In Progress tiene 3, no mover otra hasta terminar una
2. **Una tarea a la vez** - Enfocarse en completar, no en empezar
3. **Review antes de Done** - Siempre revisar antes de marcar completo
4. **Actualizar daily** - Mantener el board actualizado

## Tips

- Tareas pequenas (< 2 dias)
- Descripciones claras
- Definition of Done clara
- Blockers visibles

## WIP

Work In Progress = tareas en curso

## Metricas

- Lead time: tiempo desde que se crea hasta que se completa
- Cycle time: tiempo desde que empieza hasta que termina
- Throughput: tareas completadas por semana
