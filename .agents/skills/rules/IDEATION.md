# Ideacion - Generacion de Ideas

## Principios

1. **Cantidad antes que calidad** - Generar muchas ideas primero
2. **Sin juicio** - No critiquear ideas durante la generacion
3. **Ideas salvajes bienvenidas** - Las ideas locas pueden levar a soluciones innovadoras
4. **Combinar y mejorar** - Mezclar ideas existentes

## Proceso

### 1. Definir el problema
- Que necesidad hay?
- Que problema resuelve?
- Quien es el usuario?

### 2. Investigar
- Como lo resuelven en otras industrias?
- Cuales son las mejores practicas?
- Que tecnologias nuevas existen?

### 3. Generar ideas
- Brainstorming
- SCAMPER
- Thinking hats
- reverse thinking

## Tecnicas

### SCAMPER

| Letra | Significado | Pregunta |
|-------|------------|----------|
| S | Substitute | Que puedo sustituir? |
| C | Combine | Que puedo combinar? |
| A | Adapt | Como puedo adaptar? |
| M | Modify | Que puedo modificar? |
| P | Put to other use | Que otro uso le doy? |
| E | Eliminate | Que puedo eliminar? |
| R | Reverse | Que puedo revertir? |

### Cross-Industry

Buscar soluciones en otras industrias:

| Industria | Solucion | Aplicar a |
|-----------|----------|-----------|
| Hospitales | Triaje | Tickets IT |
| Aviones | Black box | Logging |
| Restaurantes | Menu del dia | API versioning |
| Juegos | Logros | Gamification |
| Netflix | Recomendaciones | Busqueda |

### Reverse Thinking

En vez de "como hacer X", pensar "como NO hacer X" o "que pasaria si no existiera X?"

## Herramientas

- Mind maps
- Post-its
- whiteboards
- Diagrams

## Reglas para generacion

1. **Primero entender** el problema antes de generar soluciones
2. **Investigar** que existe antes de inventar
3. **No reinventar la rueda** - Usar soluciones probadas
4. **Considerar restricciones** - Tiempo, presupuesto, tecnologia
5. **Documentar** todas las ideas aunque parezcanlocas

## Output

Del proceso de ideacion debe salir:
- Lista de opciones
- Pros/Contras de cada una
- Recomendacion
- Siguientes pasos
